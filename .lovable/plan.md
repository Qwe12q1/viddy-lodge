

## Добавление удаления видео и отключение подтверждения email

### 1. Отключить подтверждение email при регистрации

Использовать инструмент configure-auth для включения auto-confirm, чтобы пользователи могли сразу войти без подтверждения почты.

### 2. Кнопка удаления видео на странице просмотра

На странице `/watch/:id` добавить кнопку "Удалить" рядом с кнопкой "Назад", которая:
- Видна только владельцу видео (сравнение `video.user_id` с текущим `user.id`)
- При нажатии показывает диалог подтверждения (AlertDialog)
- Удаляет файл из storage: `supabase.storage.from('videos').remove([storage_path])`
- Удаляет запись из БД: `supabase.from('videos').delete().eq('id', video.id)`
- После удаления перенаправляет на главную с уведомлением

### Технические детали

**Файл: `src/pages/Watch.tsx`**
- Импортировать `useAuth`, `useNavigate`, `Trash2`, `AlertDialog` компоненты
- Добавить состояние `deleting` для блокировки кнопки
- Добавить функцию `handleDelete`:
  - Удаляет файл из storage по `video.storage_path`
  - Удаляет запись из таблицы `videos` по `video.id`
  - Навигация на `/` и toast "Видео удалено"
- Показывать кнопку удаления только если `user?.id === video.user_id`
- Обернуть кнопку в `AlertDialog` с текстом "Вы уверены? Это действие нельзя отменить"

**Auth**: Вызвать configure-auth tool с `autoConfirmEmail: true`

RLS-политики для удаления уже настроены (owner can delete), дополнительных миграций не требуется.
